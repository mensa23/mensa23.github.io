怎么计算坑爹的贷款公式

Project Orion
2017-03-21
笔记

上一次研究贷款的公式要追溯到高考时期了，那个时候的知识储备不是我吹，除了英语烂到家基本上单挑一整个美联储完全不在话下。然后现在一看这两个贷款公式直接懵逼，上礼拜 4s 店的那个销售给我算贷款的时候说的我一愣一愣的，搞得我很没面子，程序员最怕的就是没面子，尤其是在数学上。所以我要是再不写一个贷款的公式分析，都不好意思说自己是个程序员，也省的以后一天到晚查资料。

我已经把百度百科和维基百科关于贷款的词条都读了一遍了，基本上了解是怎么回事了，反正就是个很没有意思的数学游戏。

先来一点简单的。

  • 按月付息到期还本：这个一般适用于一年或一年以下的短期贷款，每个月平分总利息，然后最后一个月再加上本金。
  • 一次还本付息法：这个也是适用于一年或一年以下的短期贷款，到期一次还本付息额 = 贷款本金 * (1 + 月利率 * 贷款月数)

好了下面开始烧脑。

  • 等额本金还款：每月还款额 = (贷款本金 / 还款月数) + (本金 — 已归还本金累计额) * 月利率，首先你的贷款本金会被平分到每个月，然后用剩下的本金按照月利率计算每月的利息，两者相加就是每月应还数额。
  • 等额本息还款：每月还款额 = (贷款本金 * 月利率 * (1 + 月利率) ^ 还款月数) / ((1 + 月利率) ^ 还款月数 － 1)，其实我看到这个公式的第一眼就想放弃了，这都什么鬼。首先我们要知道理论上我们只需要在还款日还给银行 贷款本金 * (1 + 月利率) ^ 还款月数，我们先把利滚利这个事实放一放，毕竟银行也要赚钱。但是银行觉得这样不安全，毕竟万一你一下子还不出来，死皮赖脸的，会搞得大家都很难看，所以银行就想出了一个办法，让你每个月先把一笔钱放在他那里，算它借你的，那么就算到时候你还不出来，银行的损失也不会很大。那么既然是银行借你的钱，那么这些钱就会算利息。所以等到还款日那天银行借你的钱加上利息就应该是 每月还款额 + 每月还款额 * (1 + 月利率) + 每月还款额 * (1 + 月利率) ^ 2 + … + 每月还款额 * (1 + 月利率) ^ (还款月数 - 1)，然后显而易见的，这是一个等比数列求和，来来来，大家还记得等比数列求和公式么？

    化简一下就是 每月还款额 * (1 - (1 + 月利率) ^ 还款月数) / (1 - (1 + 月利率))，然后你欠银行的钱必须要和银行欠你的钱相等，所以 贷款本金 * (1 + 月利率) ^ 还款月数 = 每月还款额 * (1 - (1 + 月利率) ^ 还款月数) / (1 - (1 + 月利率))，把每月还款额提到左边，于是就成了 每月还款额 = (贷款本金 * 月利率 * (1 + 月利率) ^ 还款月数) / ((1 + 月利率) ^ 还款月数 － 1)

网上有很多等额本金和等额本息的比较，这里也整理一下。银行计算利息的方法是 利息 = 贷款本金 * 利率 * 占用时间，所以在利率固定的情况下，影响利息的因素只有贷款的金额和贷款的时长。由于等额本金每个月都会归还一部分本金，所以利率和占用时间不变的情况下，利息随着本金的减少而减少。而等额本息由于每个月给银行的钱并不是用于归还贷款，所以，利息并不会减少。这就是为什么很多人说等额本金比等额本息划算的原因。但是具体原因还是要具体分析，毕竟等额本金初期的还款额还是有点高的。

那什么时候需要贷款呢，这种仁者见仁的问题主要还是看个人，有些人觉得自己的财富增长跑得过利率，那就贷款，有些人觉得自己能跑得过通货膨胀，那就全款。我给自己的标准是如果 全款金额 ／ 总资产 > 0.618 那就贷款，《维特鲁威人》我就不贴了，影响排版美观。

现在已经是北京时间2:36了，马上天就要亮了，我的脑细胞也已经快死完了，程序员就这点不好，尤其是处女座程序员。

这篇文章就不总结了，明天问一下专业人士，没什么问题就OK了，这东西分析清楚不光自己能用说不定还能一代代往下传，别人传家书，我传笔记，简直 geek。
